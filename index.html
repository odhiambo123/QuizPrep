What is JRE

Java runtime environment, a set of components to create and run a Java application. It is a part of JDK and is made up of JVM, Class libraries and Java class loader. it provides the the programming tools and deployment technologies. JRE runs on top of the OS to provide additional Java resources

Java Development Kit, It implements the Java Language specification, JLS, and Java Virtual Machine specification, JVMS, and provide the standard edition SE of the Java Application Programming Interface.

Name Three types of exceptions

Checked Exceptions - A well written Java code should be able to recover from these exceptions
Error - these are normally external to the application and the application cannot anticipate recovering from them, eg system malfunction. Errors are not subject to Catch or specify requirement
runtime exceptions are internal to the application and the application cannot anticipate to recover from them. They may include logic errors or improper use of an API.
Errors and runtime exceptions are called unchecked exceptions

Generics

Generics Enables types (clases and interfaces) to be parameters when defining clases, interfaces and methods. It enable stronger type checks during compile time, Eliminates casting, Enables implementation of generic algorithms typesafe algorithms.

Autoboxing/Autounboxing

Compiler create objects or primitives wherever necessary during compile time. for example: converting and int to its wrapper object, integer
The Java compiler applies autoboxing when a primitive value is:
Passed as a parameter to a method that expects an object of the corresponding wrapper class.
Assigned to a variable of the corresponding wrapper class
